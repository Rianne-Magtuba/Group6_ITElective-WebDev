Database Administration

From: DBA-riyle.pdf

What is Database Administration?

Database Administration (DBA) covers the tasks, tools and practices used to maintain, secure, back up, and tune database systems so they remain available, consistent, and performant for applications and users.

DBA Responsibilities
  • Availability — ensure services are up and backups exist.
  • Accessibility — authorized users can access needed data.
  • Security — least privilege, encryption, secure configs.
  • Performance tuning — indexes, queries, resource allocation.
SQL History (short)

SQL grew from "SEQUEL" at IBM in the 1970s; it is a declarative language used to query and manipulate relational databases.

Why good schema & constraints matter

Constraints and thoughtful schema design prevent incorrect data at the source, improve query performance (via indexes), and reduce application-level validation needs.

Quick DBA Tips
  • Monitor backups and test restores regularly.
  • Log and review slow queries; add indexes where appropriate.
  • Use least-privilege accounts and rotate credentials.
  • Document schema changes and migrations.
Constraints
  • PRIMARY KEY — unique identifier for records; not NULL.
  • FOREIGN KEY — enforces referential integrity between tables.
  • UNIQUE — no duplicate values in a column.
  • NOT NULL — column must have a value on insert.
  • CHECK — restricts valid column values.
  • DEFAULT — default value when none supplied.
Keys & Indexes

Primary keys create implicit indexing for quick lookups. Composite keys use multiple columns. Indexes speed reads but add write overhead — choose wisely.

Joins: FROM+WHERE vs JOIN+ON

Older style lists tables in FROM and matches columns in WHERE. Preferred modern style is JOIN ... ON which is clearer and matches intent:

  • JOIN — use for explicit table relationships
  • WHERE — good for filtering after joining
Networking & Protocols (short)
  • HTTP / HTTPS — protocols for web resource retrieval (HTTPS adds authentication + encryption).
  • IP / TCP — underlying network addressing/transport protocols.
  • Packet routing — data travels in packets via routers across networks.
Common DBA Concepts
  • Backups & restore strategy (full/differential/log backups).
  • Replication for high availability (master/replica or clustered setups).
  • Monitoring metrics: latency, throughput, IO, locks, deadlocks.
Combining Tables — Examples

FROM + WHERE (older style):

SELECT a.column1, b.column2, c.column3
FROM table_1 a, table_2 b, table_3 c
WHERE a.table2_id = b.table2_id
  AND b.table3_id = c.table3_id;

JOIN + ON (preferred):

SELECT a.column1, b.column2, c.column3
FROM table_1 a
JOIN table_2 b ON a.table2_id = b.table2_id
JOIN table_3 c ON b.table3_id = c.table3_id;

MySQL Connection Example (full)
<?php
$DBHost = "localhost";
$DBUser = "root";
$DBPass = "";
$DBName = "tablehtmldb";

$conn = mysqli_connect($DBHost,$DBUser,$DBPass,$DBName);
if(!$conn){ die("Connection failed: ".mysqli_connect_error()); }
?>
Practical Notes
  • Always validate and sanitize inputs — use prepared statements.
  • Test restores regularly; a backup is only useful if it restores properly.
  • Indexes help SELECTs; consider write cost and storage implications.
